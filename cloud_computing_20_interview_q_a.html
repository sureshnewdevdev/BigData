Cloud Computing – 20 Interview Questions & Answers

Single‑click a question to reveal the answer. Use buttons to expand/collapse all.
1) What is cloud computing?
Cloud computing is the on‑demand delivery of IT resources—compute, storage, networking, databases, and managed services—over the internet with pay‑as‑you‑go pricing. Providers own and operate the infrastructure; customers consume it as services.
2) List the common deployment models.
Public (shared provider DCs), Private (single‑tenant), Hybrid (mix with interoperability), and Multi‑cloud (use of more than one public cloud to avoid lock‑in or optimize services).
3) Differentiate IaaS, PaaS, and SaaS.
  • IaaS: Virtualized hardware (VMs, storage, networks). You manage OS upward. Example: EC2, Azure VMs, GCE.
  • PaaS: Managed runtime and middleware. You focus on code. Example: Azure App Service, Elastic Beanstalk, Cloud Run.
  • SaaS: Complete apps delivered over the internet. Example: Office 365, Salesforce.
4) Explain the Shared Responsibility Model.
Cloud providers secure the cloud (facilities, hardware, foundational services). Customers secure what they put in the cloud (OS, apps, identities, data, configs). The exact split differs by service type—SaaS shifts more responsibility to the provider.
5) What are Regions and Availability Zones?
Regions are geographic areas containing one or more isolated data‑center groups called Availability Zones (AZs). Deploying across multiple AZs improves high availability and fault isolation.
6) CapEx vs OpEx in cloud pricing?
CapEx (capital expense) is upfront investment in hardware and facilities. OpEx (operating expense) is pay‑as‑you‑use spending on services. Cloud shifts most costs to OpEx, reducing lead time and over‑provisioning.
7) Elasticity vs. Scalability?
Scalability is the ability to handle growth (scale up/out). Elasticity is automatic scaling both up and down based on demand, optimizing cost. Auto‑scaling groups/VMSS are typical enablers.
8) What is multi‑tenancy and why does it matter?
Multi‑tenancy means many customers share the same physical resources with logical isolation. It enables economies of scale but requires strong isolation (VPC/VNet, IAM, encryption) and noisy‑neighbor controls.
9) Containers vs VMs in the cloud?
VMs virtualize hardware; each includes a full OS. Containers virtualize at the OS level, sharing the kernel and packing apps with dependencies. Containers start faster and are denser; VMs offer stronger isolation. Managed options: EKS/AKS/GKE, App Runner, Cloud Run.
10) Common cloud pricing models?
On‑demand/Pay‑as‑you‑go, Reserved/Committed use for 1–3 years (save 30–70%), and Spot/Preemptible for fault‑tolerant workloads at deep discounts. Watch storage classes and data egress fees.
11) What is a VPC/VNet?
A logically isolated virtual network in a provider’s cloud with subnets, route tables, and security groups/NSGs. It lets you run private workloads, connect via VPN/ExpressRoute/Direct Connect, and control traffic.
12) High availability vs fault tolerance?
High availability minimizes downtime via redundancy and failover across AZs. Fault tolerance continues operation with zero interruption when components fail—usually requires active‑active and higher cost.
13) What is serverless?
A compute model where you deploy functions or containers and the platform auto‑scales and bills per execution time/requests. No server management. Examples: AWS Lambda, Azure Functions, Cloud Functions.
14) Object vs block storage—when to use which?
Object storage (S3/Blob/GCS) for unstructured data, massive scale, versioning, lifecycle, cheap + eventual consistency. Block storage (EBS/Managed Disks/Persistent Disks) for low‑latency attached volumes to VMs and databases.
15) What are common cloud security challenges?
Misconfigurations (public buckets, open security groups), weak IAM, unencrypted data, secret leakage, and supply‑chain vulnerabilities. Mitigate with least privilege, CIS benchmarks, KMS‑managed keys, secret managers, and continuous posture tools.
16) Define RTO and RPO with an example.
RTO (Recovery Time Objective) is max acceptable downtime; RPO (Recovery Point Objective) is max acceptable data loss window. Example: RTO 15 min, RPO 5 min → design cross‑AZ failover and 5‑minute replication.
17) How does IAM enable least privilege?
IAM grants fine‑grained permissions to users, groups, and service principals using roles/policies. Start with deny‑by‑default, give only required actions on specific resources, and use short‑lived credentials and MFA.
18) What drives data egress cost and how to reduce it?
Cross‑region and internet outbound traffic incur charges. Reduce by caching/CDN, keeping compute close to data, private peering, compressing, and minimizing chatty cross‑cloud architectures.
19) When to choose AWS vs Azure vs GCP?
AWS: breadth + maturity Azure: Microsoft ecosystem & enterprise GCP: data/ML & global network
Real choice depends on existing contracts, skills, required services (e.g., AD/AAD integration → Azure; BigQuery/Vertex AI → GCP; widest marketplace/options → AWS), and regional presence.
20) What does a typical multi‑AZ web app architecture look like?
Clients → CDN/WAF → Load Balancer → autoscaling App tier in multiple AZs → Managed DB with read replicas → Object Storage for static assets → Queue/Pub‑Sub for async work → Monitoring + IaC for repeatable deploys.
